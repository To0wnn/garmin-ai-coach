# Garmin AI Coach

Daily (and an in-depth weekly review on Sundays) training advice for running
and cycling, based on your own Garmin Connect data, generated by Claude (or
Antigravity/Gemini) and posted to Discord — plus a web dashboard to see it
all, chat with the coach, and manage accounts.

Runs on your own Claude subscription (Pro/Max) or Gemini CLI — no separate
API costs. Owns its Garmin login and data directly (SQLite) — no InfluxDB or
other third-party sync service required.

Multi-user: each person gets their own Garmin login, own data, own advice
history, own dashboard account. An admin invites new users with one-time
links; a user without their own Claude/Antigravity subscription can borrow
another user's already-logged-in AI session via a share code.

## Requirements

- A Garmin Connect account
- A Claude Pro/Max/Team subscription (or a Gemini/Antigravity account)
- A Discord webhook URL

## Installation

```bash
git clone <repo-url> garmin-ai-coach
cd garmin-ai-coach
cp .env.example .env
```

Fill in `.env`:

- `INITIAL_ADMIN_USERNAME` / `INITIAL_ADMIN_PASSWORD` — creates the first
  (admin) dashboard account on first boot. Required.
- `DISCORD_WEBHOOK_URL` — Discord → Server Settings → Integrations →
  Webhooks → New Webhook → pick the channel → copy the URL. Used as the
  admin account's first-run default; editable per-user afterward.
- `LANGUAGE` — the language advice is written in (e.g. `English`,
  `Nederlands`, `Deutsch`, `Español`). Defaults to English.
- `WATCH_DEVICE` — exact device name as reported by Garmin (e.g. `fenix 8 -
  47mm, AMOLED`). Defaults to that same fenix 8 — only matters if your watch
  reports a different name and some field turns out device-specific.
- `LOCAL_TZ` / `DASHBOARD_PORT` — optional, sensible defaults provided.
- `CLAUDE_CODE_OAUTH_TOKEN` — generate with:

```bash
docker compose run --rm --entrypoint claude garmin-ai-coach setup-token
```

Then:

```bash
docker compose up -d
```

Open `http://<host>:4874`, log in with the admin username/password from
`.env`, then from the dashboard's **Settings** page:

1. **Garmin tab** — connect your Garmin Connect account (email/password +
   MFA code if enabled), then choose how far back to import history.
2. **General tab** — log in to Claude Code or Antigravity CLI (paste-a-code
   OAuth flow, same as the `CLAUDE_CODE_OAUTH_TOKEN` path but interactive;
   only needed if you didn't set the token in `.env`).

From here on everything runs on its own, once a day at whatever time you set
in Settings (default 06:00, your local timezone).

## Multi-user

- **Admin → Settings → Admin tab**: generate a one-time invite link, send it
  to whoever you want to add. They set their own username/password and get
  their own Garmin login, data, and advice — fully isolated from every other
  account.
- **Share codes** (AI-session sharing, not data sharing): if someone doesn't
  have their own Claude Pro/Antigravity account, the session owner generates
  a reusable share code; the borrower redeems it and their daily
  advice/chat runs through the owner's already-logged-in AI session instead
  of needing their own. Their Garmin data and dashboard stay entirely their
  own — only the AI *processing* is borrowed. Revocable at any time.

## What you get

- **Daily**: short status update + concrete advice per sport (workout type, duration, target heart rate/pace)
- **Sunday**: in-depth weekly review with trend comparison (this week vs. last week vs. 4-week average)
- Takes sleep (hours + Garmin's sleep score), training load (ACWR + per-sport load ramp), HRV/resting-HR baseline deviation, recent training history (14 days), VO2max trend, heart-rate-zone intensity distribution (are easy days actually easy?), and whether you've already trained that day into account
- Remembers its own past advice (persisted log) to stay consistent day to day instead of starting from zero every run
- Evidence-based: every recommendation references a specific number, not vague statements
- **Web dashboard** at `http://<host>:4874` — latest advice, advice history, metric charts, recent activities, an adherence timeline showing how closely you followed each day's plan (color-coded per sport, comparing planned vs. actual duration/heart rate), and a chat panel for ad-hoc follow-up questions to the coach. A "Run now" button triggers an ad-hoc advice run on demand instead of waiting for the scheduled time.
- **Settings page** (`/settings`): general preferences, Garmin connection/backfill, password change, and (for admins) user management.

## How it works

A permanent AI CLI session (Claude Code or Antigravity) runs inside the
container per AI-session owner (via tmux) — a dispatcher checks every 5
minutes whether any user's configured advice time has arrived and, if so,
sends that session a prompt with the user's pre-computed Garmin numbers. The
AI writes the advice to a file, which gets posted to Discord. After every run
the session is reset (`/clear`) so context doesn't build up over time. Garmin
data syncs into a local SQLite database every 5 minutes — no third-party
sync service in the loop.
